# Security Remediation Plan — kri-local-rag

> Generated 2026-06-23 by a full security review (first-party code SAST + live CI/security-tool results).
> Trust model: local-first, single-user, services bound to `127.0.0.1`. Several findings are severity-reduced
> by that model but become exploitable if the app is shared, ports are exposed, or documents come from third parties.

**Gap (diagnosed 2026-06-23):** Three bots open dependency PRs — Renovate (~25 security PRs), Dependabot security updates (grouped uv/github-actions PRs #103, #116), and a stray Snyk PR (#100). But the open-alert count (147 → **26** at diagnosis) does **not** fall when their PRs merge. Root cause: *both* Renovate and Dependabot security PRs edit the **autogenerated** `requirements.txt`, never `pyproject.toml`. `requirements.txt` is `uv export`ed from `uv.lock` (`make export-reqs`), so a merged bump is reverted the next time it regenerates, and the Dependabot *alert* (which reads the resolved graph) never clears. Note "Dependabot" spans two layers: the **alert scanner** (the Security-tab scorecard, always on) and **security-update PRs** (the bot) — only the latter competes with Renovate. **Reconcile first** (see Phase 0) — picking one updater is necessary but *not sufficient*; the chosen updater must edit the source of truth (`pyproject.toml`/`uv.lock`), or the list regenerates weekly.

## Part A — Dependency vulnerabilities (Dependabot, 147 open)

`requirements.txt` is **autogenerated** from `pyproject.toml` (see `.claude/rules/python-imports-and-deps.md`). Do **not** hand-edit it. The fix path is: bump the direct dep in `pyproject.toml` (or add a constraint/override for a transitive dep), then `make export-reqs`, then `uv sync`.

### A1 — Critical (4) — fix first

| Package | Vulnerable | Fixed in | Advisory | Issue |
|---------|-----------|----------|----------|-------|
| `authlib` | ≤ 1.6.8 | **1.6.9** | GHSA-wvwj-cvrp-7pv5 | JWS/JWK header injection → signature-verification bypass |
| `nltk` | ≤ 3.9.2 | **3.9.3** | GHSA-7p94-766c-hgjp | Zip-Slip (arbitrary file write on extract) |
| `unstructured` | ≤ 0.18.17 | **0.18.18** | GHSA-gm8q-m8mv-jj5m | Path traversal via malicious MSG attachment → arbitrary file write |
| `langchain-core` | < 0.3.81 | **0.3.81** | GHSA-c67j-w6g6-q2cm | Serialization injection → secret extraction in `dumps`/`loads` |

`unstructured` is a **direct** dep (`unstructured[pdf]>=0.18.15,<0.19.0`) — bump the floor to `>=0.18.18`. `authlib`, `nltk`, `langchain-core` are transitive — bump the parents (`langchain`/`langchain-community`) and/or pin a minimum via `[tool.uv]` constraints. These two (`unstructured` MSG path-traversal, `langchain-core` secret extraction) directly touch this app's ingestion + LangChain usage — highest real-world risk.

### A2 — High (46) — notable packages

Bump these floors in `pyproject.toml` (direct) or via constraints (transitive). Highest-impact for a RAG/web app:

- **`langchain` → ≥ 0.3.30** (GHSA-3644-q5cj-c5c7) — direct dep, used in `qa_loop.py`/ingestion.
- **`urllib3` → ≥ 2.7.0** (GHSA-qccp-gfcp-xxvc) — transitive via requests/httpx.
- **`cryptography` → ≥ 48.0.1** (GHSA-537c-gmf6-5ccf).
- **`tornado` → ≥ 6.5.6** (3 Highs) — pulled in by Streamlit.
- **`pillow` → ≥ 12.2.0** (GHSA-pwv6-vv43-88gr) — direct dep; image parsing in PDF backends.
- **`python-multipart` → ≥ 0.0.30** — Streamlit upload path.
- **`GitPython` → ≥ 3.1.50**, **`lxml` → ≥ 6.1.0**, **`pyarrow` → ≥ 23.0.1**, **`langsmith` → ≥ 0.8.18**, **`nltk`**, **`onnx`**, **`orjson`**, **`protobuf`** (direct, `>=6.32.0`), **`pyasn1`**.

### A3 — Medium (73) / Low (24)

Bulk-resolved by the same `pyproject.toml` floor bumps + `make export-reqs`. Notable: `aiohttp` (10 alerts → ≥ 3.14.1+), `pypdf` (9 → ≥ 6.13.3), `authlib`, `idna` → ≥ 3.15, `requests`, `transformers`, `streamlit`, `python-dotenv` → ≥ 1.2.2, `marshmallow`, `virtualenv`, `pytest`, `Pygments`, `fonttools`, `filelock`.

### Dependency remediation procedure

```bash
# 1. Edit pyproject.toml: raise direct-dep floors (unstructured, langchain*, pillow, protobuf, requests...)
#    For transitive-only CVEs add a [tool.uv] constraint/override pinning the minimum safe version.
# 2. Resolve + regenerate the lockfile and requirements.txt
uv sync
make export-reqs
# 3. Verify nothing regressed
.venv/bin/python -m pytest tests/unit tests/integration
make pip-audit          # should report 0 actionable
# 4. Confirm against the source of truth
gh api repos/KristjanHS/kri-local-rag/dependabot/alerts --paginate \
  -q '[.[] | select(.state=="open")] | length'
```

Do this in batches (Critical → High → Medium/Low), running tests between batches so a breaking major bump is isolated.

## Part B — First-party code findings (SAST review)

### B1 (HIGH) — Stored/reflected XSS via `unsafe_allow_html`
`frontend/rag_app.py:132-135`, `:147-150`. LLM output (derived from ingested documents + the user question) is interpolated into a raw HTML string rendered with `unsafe_allow_html=True`. A document containing `<img src=x onerror=...>` → script execution in the viewer's browser (indirect/RAG markup injection).
**Fix:** render answer text with `st.markdown(text)` *without* `unsafe_allow_html`, or `html.escape(...)` the content before interpolating into the `data-testid` wrapper.

### B2 (HIGH) — Path traversal in PDF upload handler
`frontend/rag_app.py:51-55`. `path = os.path.join(save_dir, f.name)` uses the client-supplied filename unsanitized; an absolute path or `../` escapes `data/` (and `os.path.join` with an absolute component discards `save_dir`).
**Fix:** `name = os.path.basename(f.name)`, validate extension allowlist, then assert `os.path.realpath(path).startswith(os.path.realpath(save_dir) + os.sep)` before opening.

### B3 (MEDIUM) — No size/count/type limits on ingestion (DoS)
`frontend/rag_app.py:42-55`, `backend/ingest.py:61-137`. Unbounded file count/size written to disk then embedded; extension check is client-side only, `ingest.py` keys off extension not content.
**Fix:** enforce max upload size + count in UI; validate magic bytes (runtime already ships `libmagic1`); cap total ingested bytes per request.

### B4 (MEDIUM) — Weaviate anonymous access, no auth
`docker/docker-compose.yml:13` (`AUTHENTICATION_ANONYMOUS_ACCESS_ENABLED: 'true'`). Bound to `127.0.0.1` → Medium. Any local process can read/delete the index.
**Fix:** acceptable for strict single-user localhost — **document the assumption**. If multi-user or port ever bound to `0.0.0.0`, enable API-key auth (a ready `docker/weaviate_auth.notused` exists to wire in).

### B5 (MEDIUM) — `auto-pr.yml` missing least-privilege permissions
`.github/workflows/auto-pr.yml`. No top-level `permissions:` block → inherits broad default `GITHUB_TOKEN`. Trigger is `workflow_dispatch` only and `${{ github.ref_name }}` is an action input (not a `run:` shell), so not directly injectable, but the missing least-privilege block is a real gap (other workflows already set one).
**Fix:** add `permissions: { contents: read, pull-requests: write }`.

### B6 (LOW) — Unpinned third-party Action tags
`meta-linters.yml`: `docker://rhysd/actionlint:latest`, `hadolint/hadolint-action@v3.1.0`; `auto-pr.yml`: `peter-evans/create-pull-request@v6`. Mutable tags = supply-chain risk.
**Fix:** pin third-party actions to a full commit SHA (especially `:latest`).

### B7 (LOW) — Mutable base-image tags in compose
`docker/docker-compose.yml` — `ollama/ollama:latest`, Weaviate image tag-pinned not digest-pinned (`app.Dockerfile` already documents digest-pinning as the goal). App Dockerfile correctly runs as non-root `appuser`.
**Fix:** pin `ollama`/`weaviate` images by `@sha256:` digest.

### B8 (LOW) — CI tool binaries downloaded without checksum verification
`scripts/ci/install-system-tools.sh:72-141`. hadolint/actionlint/yamlfmt fetched from `releases/latest` with only an ELF sanity check, no checksum/version pinning. Dev tooling only.
**Fix:** pin versions (the `*_VERSION` envs already support it) and verify SHA256 against published `checksums.txt`.

### B9 (LOW) — Save dir relative to CWD
`frontend/rag_app.py:48-49` writes to relative `"data"`. Canonicalize alongside B2.

**Confirmed clean:** no SQL/NoSQL injection (typed Weaviate client + parameterized filters), no `shell=True`/`os.system`, no unsafe deserialization (`pickle`/`yaml.load`/`eval`/`exec`/`torch.load`/`joblib`), no `verify=False`, no SSRF (HTTP targets are config-resolved env/localhost only), no secrets logged or hardcoded, no `pull_request_target`, no `${{ github.event.* }}` in `run:` blocks. Shell scripts use `set -euo pipefail` + quoted expansions; destructive `down -v` gated behind confirmation/test profiles.

## Remediation plan (artifact-grouped execution order)

**Ordering principle:** grouped by *artifact*, not by severity. Each file is edited exactly once
(landing in final form), and each expensive command — `uv sync`, `make export-reqs`, full `pytest` —
runs once instead of per-severity-batch. Severity still drives *what* gets fixed (Part A/B above); it
just no longer drives *sequencing*, because all 147 dep alerts resolve through the same
`pyproject.toml`/lockfile and findings B1/B2/B3/B9 all live in `frontend/rag_app.py`.

**Two ordering decisions (reduce debugging cost):**
1. **Deps (Phase A) land before code (Phase B).** Independent files, but getting tests green on the
   dep bump first means a later code-edit failure can't be confounded by a dep-induced one.
2. **Deps in one shot; bisect only on failure.** Most bumps are minor/patch within existing `<X.0.0`
   caps (near-zero break risk); one `uv sync` surfaces any resolution conflict atomically. Pre-flag the
   only real break candidates — major crossings `cryptography`→48, `pyarrow`→23, `pillow`→12 — instead
   of pre-splitting all 147.

### Phase 0 — Reconcile updaters (config + CI only, no app code — do first)

**Decision (2026-06-23):** Renovate is the single updater. Dependabot stays as the *alert scanner* only (the Security-tab scorecard). Snyk is removed. Renovate is kept because it also delivers the Action SHA-pinning (B6) and image digest-pinning (B7) this plan wants (`helpers:pinGitHubActionDigests` + `docker:pinDigests`), which Dependabot cannot do, and gives one noise-control surface (grouping, schedule, `minimumReleaseAge`) across python + actions + docker.

- [ ] **Disable Dependabot security-update PRs, keep alerts:** `gh api -X DELETE repos/KristjanHS/kri-local-rag/automated-security-fixes` (leaves `vulnerability-alerts` / the Security tab — the scorecard — untouched). Then close the duplicate Dependabot PRs #103 (uv group) and #116 (github-actions group).
- [ ] **Remove Snyk:** close PR #100 and uninstall the Snyk GitHub App (one-off; not part of the chosen toolchain).
- [ ] **Repoint Renovate at the source of truth** (`renovate.json`): swap `enabledManagers` `pip_requirements` → `pep621` (reads `[project]` deps in `pyproject.toml`; auto-detects uv from `uv.lock` and runs `uv lock`); delete the `pip_requirements` block (fileMatch `requirements*.txt`); switch every `matchManagers: ["pip_requirements"]` packageRule to `["pep621"]`; add `"lockFileMaintenance": { "enabled": true }`.
- [ ] **Make `requirements.txt` a CI-derived artifact, not a committed file.** It is consumed only by `trivy_pip-audit.yml` / `make pip-audit` (which already regenerates it) and is **not** used by the Docker build. The hosted Renovate App cannot run `postUpgradeTasks` (`make export-reqs`), so don't fight to keep the derived file in sync — stop committing it: `git rm --cached requirements.txt`, add to `.gitignore`. In `trivy_pip-audit.yml`, change the path trigger `requirements.txt` → `uv.lock` + `pyproject.toml`, and add a `make export-reqs` step before the Trivy/pip-audit scan. *(Aligns with the `python-imports-and-deps` rule: `requirements.txt` is autogenerated.)*

**Acceptance:** a Renovate security PR modifies `pyproject.toml` (and `uv.lock`), not `requirements.txt`; after merge the matching Dependabot alert auto-closes within a scan cycle and does **not** reappear on the next dep change. *(Supersedes the old "confirm it opens PRs" criterion — opening PRs was never the failure; non-sticking merges were.)*

### Phase A — `pyproject.toml` → `uv.lock` → `requirements.txt` (one pass, all 147 deps)
- [ ] In a **single** `pyproject.toml` edit: raise all direct-dep floors (`unstructured`≥0.18.18, `langchain*`, `pillow`≥12, `protobuf`, `requests`, …) and add `[tool.uv]` constraints/overrides for transitive CVEs (`authlib`≥1.6.9, `nltk`≥3.9.3, `langchain-core`≥0.3.81, `urllib3`≥2.7.0, `cryptography`≥48.0.1, `tornado`≥6.5.6, `aiohttp`, `pypdf`≥6.13.3, `idna`≥3.15, `python-dotenv`≥1.2.2, GitPython, lxml, pyarrow, orjson, pyasn1, transformers, marshmallow, virtualenv, fonttools, filelock, Pygments).
- [ ] `uv sync` → `make export-reqs` (once each). On resolution conflict, relax the offending pin.
- [ ] Targeted tests; **bisect only if they fail** (suspect the 3 major crossings first). Commit.

### Phase B — Application code (grouped by file)
- [ ] `frontend/rag_app.py` — one read/edit pass covering **B1** (drop `unsafe_allow_html` / `html.escape` LLM output), **B2 + B9** (`basename` + realpath containment on upload), **B3-frontend** (size/count caps).
- [ ] `backend/ingest.py` — **B3-backend** (magic-byte validation, per-request byte cap).
- [ ] Run targeted frontend/ingestion tests once. Commit.

### Phase C — Infra & CI hardening (grouped by file; no app tests)
- [ ] `docker/docker-compose.yml` — **B4** (document localhost/single-user assumption or wire `weaviate_auth.notused`) + **B7** (`@sha256:`-pin `ollama`/`weaviate` images).
- [ ] `.github/workflows/auto-pr.yml` — **B5** (least-privilege `permissions:`) + **B6** (SHA-pin `peter-evans/create-pull-request`).
- [ ] `.github/workflows/meta-linters.yml` — **B6** (SHA-pin `rhysd/actionlint`, `hadolint-action`).
- [ ] `scripts/ci/install-system-tools.sh` — **B8** (pin versions + SHA256-verify downloads).
- [ ] actionlint/hadolint lint pass. Commit.

### Final verification (once, after all phases)
- [ ] `.venv/bin/python -m pytest tests/unit tests/integration` + `make pre-commit` green.
- [ ] `make pip-audit` = 0 actionable; re-query open Dependabot count (`gh api … | length`); CodeQL/Semgrep still 0 open.
